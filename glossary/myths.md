# Networking Myths

- Myth: If the application is slow, the server is always the cause. Reality: latency, loss, DNS, TCP behavior, routing, and wireless retransmissions can all surface as application slowness.
- Myth: Bandwidth and latency are the same. Reality: bandwidth is capacity; latency is delay. High capacity does not remove propagation, queuing, or protocol round trips.
- Myth: Encryption solves all network security problems. Reality: encryption protects confidentiality, but authentication, integrity, endpoint security, key management, and operations still matter.
