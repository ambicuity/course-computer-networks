# Code Division Multiplexing to The Politics of Telephones

> CDMA (Code Division Multiple Access) lets every station transmit across the **entire** band, all the time, by spreading each bit over `m` short **chips** (typically 64 or 128 in real systems; `m = 8` in Tanenbaum's worked example). Each station gets a unique **chip sequence**: it sends the sequence for a 1 bit, the negated sequence for a 0 bit, or silence. On the shared medium the bipolar (+1/-1) signals **add as voltages**, and a receiver recovers one station by computing the **normalized inner product** `S . C = (1/m) Σ Sᵢ Cᵢ`. The codes are **pairwise orthogonal** (Walsh codes generated by the Hadamard recursion), so foreign stations contribute exactly 0 and cancel; the target's own term is +1, -1, or 0. With `2ⁿ` Walsh codes of length `2ⁿ` you get `2ⁿ` channels. The catch — chip-level time synchronization — holds in this idealized model but breaks in cellular, which is why IS-95 cdmaOne (1990s) and WCDMA use a different asynchronous design. The lesson's back half traces the same "shared resource, who owns it" question into the PSTN: local loops, end offices, toll/tandem offices, intertoll trunks, and the regulatory plumbing the 1984 AT&T breakup forced into the network — the MFJ, 164 LATAs, LECs, and IXCs reachable only through a POP.

## The Problem

A wireless team is debugging a CDMA-style uplink where two handsets keep clobbering each other. The vendor swears "CDMA means everyone can talk at once on the same frequency, no scheduling needed." So why does adding a third active handset suddenly corrupt the second one's bits?

The answer is not that the band ran out. It is that recovery depends on a single fragile property — **orthogonality of the chip codes plus chip-level synchronization**. When codes are truly orthogonal and chips line up in time, the math is clean: foreign stations cancel to zero. Break either assumption — reuse a non-orthogonal code, or let one transmitter drift half a chip late — and the cancellation stops being exact. The correlation lands somewhere between -1 and +1, and the decoder can no longer tell a 1 from a 0. To diagnose this you have to be able to do the receiver's arithmetic yourself.

The same "who owns the shared thing" question reappears one layer up in the telephone network, where the answer was written by a 1984 court order.

## The Concept

### Three ways to share one channel

FDM, TDM, and CDM are three answers to one question. The airport-lounge analogy from the chapter makes the difference concrete:

| Scheme | How it shares | Lounge analogy |
|---|---|---|
| **TDM** (Time Division) | Each station gets the whole band, but only during its time slot | Pairs take turns speaking |
| **FDM** (Frequency Division) | Each station gets a slice of the band, all the time | Pairs speak at different pitches |
| **CDM / CDMA** (Code Division) | Every station uses the *whole* band *all* the time, separated by code | Everyone talks at once, each pair in a different language |

CDMA is a **spread spectrum** technique: a narrowband signal is deliberately smeared across a wider band, which buys interference tolerance and lets multiple users coexist. Because the multi-user case dominates, CDM is usually called CDMA.

### Chips, chip sequences, and the bandwidth cost

Each bit time is sliced into `m` short intervals called **chips**. Real systems use 64 or 128 chips per bit; the worked example uses `m = 8`. Each station is assigned a unique `m`-chip **chip sequence**, written in bipolar form as a vector of -1 and +1. For example, station A might hold:

```
A = (-1 -1 -1 +1 +1 -1 +1 +1)
```

The encoding rule has exactly three legal outputs per bit time: `1` sends the chip sequence (A), `0` sends its negation (`A`-bar), and a silent station sends all zeros. Spreading a `b` bit/s stream into `m·b` chips/s widens the required bandwidth by a factor of `m`. The chapter's numbers: a 1-MHz band shared by 100 stations gives each 10 kHz / 10 kbps under FDM; under CDMA each station uses the **full 1 MHz** at 100 chips per bit to spread its 10-kbps stream across the whole channel.

### Orthogonality is the whole trick

Codes are chosen to be **pairwise orthogonal**: the normalized inner product of any two distinct sequences is zero.

```
S . T  ≡  (1/m) Σᵢ Sᵢ Tᵢ  =  0      for S ≠ T
S . S  =  (1/m) Σᵢ Sᵢ²    =  1      (every term is (±1)² = 1)
S . S-bar = -1
```

"In plain English, as many pairs are the same as are different." This is generated systematically with **Walsh codes** — the rows of a Hadamard matrix. `code/main.py` builds them with the recursion `H(n) = [[H, H], [H, -H]]`, giving `2ⁿ` orthogonal codes of length `2ⁿ`, and asserts the orthogonality property before running any simulation.

### Superposition on the wire

All stations are assumed **synchronized in time**, so every chip sequence starts at the same instant. When several stations transmit at once, their bipolar sequences **add linearly** — three +1 chips and one -1 chip in the same chip period yield +2 (think 3 V minus 1 V = 2 V on the medium). If B and C both send 1 bits:

```
B = (-1 -1 +1 -1 +1 +1 +1 -1)
C = (-1 +1 -1 +1 +1 +1 -1 -1)
S = B + C = (-2  0  0  0 +2 +2  0 -2)
```

The SVG (`assets/code-division-multiplexing-to-the-politics-of-telephones.svg`) shows this for `m = 4`: four stations encode their bits, the chips sum into a single channel vector, and the receiver correlates that vector against one station's code.

### Recovery: correlate and cancel

To pull one station's bit out of the sum, the receiver computes the normalized inner product of the received vector `S` with that station's known code `C`. Suppose A and C send 1 while B sends 0; the receiver sees `S = A + B-bar + C` and computes:

```
S . C = (A + B-bar + C) . C = A.C + B-bar.C + C.C = 0 + 0 + 1 = 1
```

The first two terms vanish **because the codes are orthogonal** — this is exactly why orthogonality is mandatory. A result of `+1` means a 1 bit, `-1` means a 0 bit, and `0` means the target was silent. `code/main.py` runs six such scenarios (only C active, B+C, a three-way collision, all four active) and decodes every one correctly. Run it and watch `S . C = +1 -> bit 1` for the colliding case.

For the collision `A=1, B=0, C=1` with `m = 4` Walsh codes, `main.py` prints `S = +1 +3 -1 +1`; correlating against C's code `+1 +1 -1 -1` gives products `+1 +3 +1 -1`, summing to `+4`, divided by `m` = `+1` -> C sent 1. A, B, and D drop out cleanly. Reuse a non-orthogonal code and the cross terms stop being zero, the correlation lands off the integers, and `recover_bit()` raises rather than guessing.

### Where the ideal model breaks

Two assumptions carry the whole construction: **orthogonal codes** and **chip-level synchronization**. With `2ⁿ` Walsh codes of length `2ⁿ`, you can grow the number of concurrent stations arbitrarily by lengthening the codes. But synchronization "is not even approximately true" in cellular, where handsets are at different distances and arrive with different delays. CDMA was deployed in cellular starting in the 1990s (IS-95 cdmaOne, later WCDMA) using an **asynchronous** design that tolerates misalignment, plus satellites and cable networks. The synchronous model here is the teaching foundation, not the cellular reality.

### The politics: from chips to circuits

The telephone network answers the same shared-resource question with a hierarchy. Each phone has two copper wires running 1-10 km to its **end office** (local central office) — about 22,000 in the US — over the **local loop**, the last analog twisted-pair link in an otherwise digital system. A call within one end office is a direct loop-to-loop connection. Across offices it climbs: end office -> **toll connecting trunk** -> **toll office** (or **tandem office** locally) -> high-bandwidth **intertoll trunks** -> back down. The three components: **local loops** (analog), **trunks** (digital fiber), **switching offices**.

The 1984 AT&T breakup turned regulation into network architecture. The **MFJ** (Modified Final Judgment) split AT&T into AT&T Long Lines plus 23 BOCs (grouped into 7 RBOCs). The country was carved into **164 LATAs** (Local Access and Transport Areas, roughly one area code each). Inside a LATA, one **LEC** (Local Exchange Carrier) held the local monopoly; all inter-LATA traffic went to an **IXC** (IntereXchange Carrier) reachable only through a **POP** (Point of Presence) the LEC was required to connect — on identical terms for every IXC, so a subscriber could *choose* their long-distance carrier. The boundary between "local" and "long distance" became a real, addressable structure in the switching fabric.

## Key Terms

| Term | What people say | What it actually means |
|---|---|---|
| Chip | "a bit" | One of `m` short intervals a single bit is spread across; 64-128 per bit in real systems |
| Chip sequence | "the station's code" | An `m`-element bipolar (+1/-1) vector; sent as-is for a 1, negated for a 0 |
| Orthogonal codes | "different codes" | `S . T = 0` for distinct codes — the exact property that makes foreign stations cancel |
| Walsh code | "a CDMA code" | Rows of a Hadamard matrix; `2ⁿ` orthogonal sequences of length `2ⁿ` |
| Normalized inner product | "decoding" | `(1/m) Σ Sᵢ Cᵢ`; yields +1, -1, or 0 in a clean synchronous system |
| Spread spectrum | "noise-resistant radio" | Smearing a narrowband signal over a wider band for interference tolerance and multi-access |
| Local loop | "the phone line" | The 1-10 km analog twisted pair from a subscriber to the end office — the last analog link |
| Toll / tandem office | "the phone exchange" | Second-level switch joining end offices via toll connecting and intertoll trunks |
| MFJ | "the AT&T breakup" | Modified Final Judgment, effective Jan 1, 1984; created BOCs/RBOCs and the LATA structure |
| LATA / LEC / IXC / POP | "phone company stuff" | 164 access areas, the local monopoly carrier, the inter-LATA carrier, and its access switch |
